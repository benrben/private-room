# Big-picture review — "how to become the most talked-about product of the year" (2026-07-06)

This review was made by a large team of AI research agents: 4 read the whole
codebase, 4 researched the market on the live web, 6 proposed bold ideas
(35 ideas in total), 3 "judges" attacked every idea to kill the weak ones,
and 1 wrote the final plan. This document goes beyond the 2026-07-04 review
(parts 1–7). Almost everything from that review is already built — only
HLT-2 (signing/updates) and the long-term items are still open.

---

## 1. The main idea

**Stop selling Private Room as a "chat with AI on your Mac" app. That fight
is already lost.** LM Studio, Jan, and Ollama's own app do that for free,
they look great, and everyone knows them. Even worse: Apple's next macOS
(around September) will let anyone ask questions about their files, built
into the system. In about a year, "chat with your files" will be a free
checkbox in the operating system.

But what you actually built, nobody else has:

> **Other apps' AI chats NEXT to your files. Your AI works INSIDE the lock —
> inside one encrypted file — and it can prove every answer with a real quote.**

We checked. No shipping product combines these three things: one portable
encrypted file + a local AI that can edit and mark real documents + proof
for every quote. The closest product (DEVONthink 4) on purpose blocks its
AI from touching its encrypted data.

Also, 2026 gave you free marketing — real news stories, not opinions:

- A US court ruled that **AI chats are not private in court** (US v. Heppner).
  A man's Claude chats were used against him.
- Another court forced OpenAI to **keep every ChatGPT chat forever** —
  even "deleted" ones — and hand 20 million of them to lawyers.
- **Meta bought Rewind and shut it down.** People's private AI memory
  disappeared because of a business deal.
- A popular AI agent (OpenClaw) had a security hole; **21,639 people's agents
  were exposed** on the internet.
- Claude's file agent leaked **social security numbers** through a hidden
  message inside a Word file — 48 hours after launch.

Every one of these stories says: "you need what Private Room already is."
You don't need to build the story. You need to stop hiding it.

**The new position: it's not an app, it's a file.** Private Room is a file
format that happens to come with a great app — like how KeePass is really
the `.kdbx` file, not the program. Everything should serve one sentence:

> **"Your AI works inside an encrypted file you own, and it proves every
> answer with a receipt."**

Three promises: it's **yours** (a file, not an account), it's **sealed**
(offline, and you can check that yourself), it **can't lie** (verified
quotes).

You have roughly **12–18 months** before Apple and Microsoft make the basic
version free. Spend that time making the FILE — not the chat — the thing
people can't give up.

## 2. The gate — must fix before any launch (~3–4 weeks)

Right now, out of 100 people who see the README, only about **3 to 8**
ever get a first answer from the app. Fix these first; nothing else matters
until they're done.

1. **"The Seal" — make the encryption something you can SEE and FEEL.**
   Today the product's biggest feature is invisible: encryption happens
   silently, so the app looks like any notes app. Turn locking and
   unlocking into a short, beautiful ritual that people want to
   screen-record:
   - **Unlock:** the violet keyhole glows, the "door" opens (the 700ms
     bloom from the create flow already does this), and the room's files
     light up one by one — like walking into a real room and turning on
     the lights.
   - **Seal:** when you lock, the panels visually fold into the keyhole,
     a soft, satisfying "thunk" sound plays, and the window shrinks down
     into the `.roomai` file icon — you literally SEE your whole workspace
     become one sealed file. Encryption stops being a claim and becomes a
     moment.
   - **Seal & Send:** right after sealing, the file icon slides up with an
     AirDrop button under it — sealing and handing someone the room becomes
     one gesture, the product's signature move.
   - **Why it wins:** no competitor can show their security, because their
     security is a settings page. Yours becomes the closing shot of the
     launch film, the app's App-Store screenshot, and the 5-second clip
     strangers repost. The brand (violet keyhole on ink) already exists;
     this gives it motion and sound. Cost: ~1–2 weeks, all frontend, riding
     the existing bloom animation, auto-lock machinery, and document icon.
2. **Fix the auto-updater — it is broken in three ways.**
   - The update key in `tauri.conf.json` is still the placeholder text
     `REPLACE_WITH_UPDATER_PUBLIC_KEY`.
   - It looks for updates at `benreich/private-room`, but your GitHub is
     `benrben/private-room`. RELEASING.md has the same wrong name, so
     following your own checklist keeps it broken.
   - The v0.1.0 release has no update files at all (no `.app.tar.gz`, no
     `latest.json`).
   **This means every person who installed the app can never get a fix from
   you.** Every new download today adds another person you can never reach.
3. **Remove the "install Ollama yourself" step.** Build the AI engine into
   the app (llama.cpp or MLX), the same way Whisper is already built in.
   Download the model on first run with the progress bar you already have.
   Don't advertise it — it's not a story, it's just what users expect.
   While you're in there, two quick wins:
   - **The smart search is asleep on every Mac.** You already built
     meaning-based search (embeddings). But the app never downloads the
     small model it needs (`nomic-embed-text`, ~270MB), so it silently does
     nothing on every install. One download call turns on a finished feature.
   - **Image marking is broken for new users.** "Mark the signature" needs a
     vision model, but nothing ever offers to install one. So the app falls
     back to the chat model — which your own code comment says draws the
     boxes in the wrong place. Offer the download the first time someone
     tries it.
4. **Fix the things a security expert will catch in five minutes** —
   because at launch, they will look:
   - The app's web-view security policy (CSP) is empty (`null`). Rooms can
     contain files made by strangers, so this matters.
   - The app says "End-to-end encrypted" in one place. That's the wrong
     term (it's "encrypted at rest"). Privacy people will call this out.
   - Lock down the encryption settings (SQLCipher PRAGMAs) and write which
     embedding model was used inside the file — otherwise a future version
     could silently misread old rooms.
   - **Give people a recovery sheet.** Today, forgetting your password means
     losing everything, and the only warning is one small gray line. Offer a
     printed recovery key when a room is created.
5. **Tell the story better, cheaply:**
   - The README's best line — the AI must quote for real, and the app checks
     the quote before showing it — is buried as bullet 4 of section 3.
     Put it first.
   - "AirDrop your whole workspace as one file" is hidden in the middle of a
     paragraph. Put it second.
   - The `#` commands have help text in the code that is never shown to
     users. Show it, and add a `#help` command.
   - Error messages say "go to Settings → …" but never include a button
     that takes you there. Add the buttons.

## 3. The four big bets

### Bet 1 — "Receipts": the AI that cannot fake a quote (2–3 weeks)
Make your quote-checking the star of the product. Every quote the AI shows
gets a "verified" checkmark. One click exports a PDF with the highlights
drawn in and a small line at the bottom: "Every highlight verified against
the source. Made offline in Private Room."

- **Why it wins:** your app checks the AI's quotes with code, at runtime.
  No competitor can copy that with a clever prompt — they'd have to rebuild
  their product. And after the 2026 court stories about fake AI citations,
  people are primed for it.
- **The demo:** ask the same trick question to another local AI app and to
  Private Room. Theirs gives a confident wrong summary. Yours jumps to the
  exact sentence — or says "that's not in the document."

### Bet 2 — "The First Room": the installer is the show (1–2 weeks, mostly writing)
Ship a locked demo room inside the DMG, **with its password printed right on
the DMG background picture** — the key left under the mat on purpose.
Inside: a short fictional case with a small mystery. Every step runs on the
reliable paths (pre-checked highlights, `#extract`, click-a-line-to-play
audio). Label it honestly as a "guided replay", with a "now ask it live"
button once the real model finishes downloading.

- **Why it wins:** it fills the empty 20 minutes after install, it gives
  every reviewer a perfect first demo, and the screenshot of a password
  printed on the box spreads by itself. Your room-templates code already
  does most of the work.

### Bet 3 — "A room is a file": promise + file spec + small tool (2–3 weeks)
One beautiful web page with a signed promise: **no account, no server, no
telemetry, ever — and if this project dies, your rooms still open.** Under
it: the published `.roomai` file format description, and a tiny open-source
command-line tool (`roomai verify` / `roomai export`) built from the same
Rust code the app uses.

- **Why it wins:** the two questions everyone will ask are "can I trust
  crypto from one unknown developer?" and "what if you disappear?" This
  turns both attacks into your headline. Obsidian proved the promise page IS
  the marketing. After Rewind died, "a file can't be bought and shut down"
  hits hard.
- **The demo:** run `strings room.roomai | grep client` — you get gibberish.
  Run `roomai export` with the password — you get everything.
- **Careful wording:** say "there is no us to subpoena" — never
  "subpoena-proof". Lawyers read these pages.

### Bet 4 — "The Leash": the room as a plug for other AIs (2–3 weeks, second wave)
You already built 90% of this (`room_mcp.rs`). Add one switch: an unlocked
room becomes an MCP server — the private memory that Claude Desktop, Cursor,
and any AI agent can read. Local programs only by default; cloud programs
behind a clear warning ("what they read, they keep"). Each program needs
your approval once, using the approval system you already have.

- **The video clip:** Cursor quotes from your encrypted vault, offline. You
  lock the room with Touch ID. The agent's next request dies. Five seconds.
- **Why it wins:** after the OpenClaw security mess, "AI agents need a
  memory with a door that locks" is the message of the moment. And every
  MCP tool directory that lists you is free advertising you couldn't buy.

## 4. The launch — one planned event, three steps

**The film:** one unbroken 60-second recording at a desk. Wi-Fi turned OFF
on camera, a network monitor visible and silent. Double-click The First
Room, type the password printed on the DMG art, ask about the contract, the
receipt chip jumps to the exact clause, dictate a note by voice, seal the
room (make it feel and sound good — that's the closing shot), AirDrop the
file. Practice it with your mock-Ollama test setup so it's perfect every
take. Put a "check for yourself that it never phones home" script in the
README.

- **Step 1 — friendly crowd (week 0):** post to r/LocalLLaMA and r/ollama,
  and get listed in Ollama's integrations. These people already have Ollama
  installed, so your biggest friction doesn't exist for them. Fix the bugs
  they find.
- **Step 2 — Show HN (week 1–2):** *"Show HN: I put an AI workspace inside
  one encrypted SQLite file."* The link goes to the promise page; the thing
  to try is The First Room. The file spec and the little tool go public the
  same day, so "is it open source?" already has an answer. In your maker
  comment: explain the small-model engineering (constrained decoding,
  stable prompts, tool routing — Hacker News loves this) and admit one real
  limitation plainly.
- **Step 3 — The Leash (weeks 4–6):** release the MCP switch with a short
  security write-up ("agents need bodies with doors") and submit it to the
  MCP tool lists.
- **Always ready, no date:** a comparison page ("what a court can get from
  ChatGPT vs. from a .roomai file") and the gibberish-grep clip, ready to
  post the same day the next AI privacy scandal breaks. Signal grew 43x in
  one week off someone else's scandal. Use real events only — never fake
  outrage (the company that tried that imploded).

## 5. What keeps it alive after the noise

- **The file keeps people.** Because the spec and tool mean "you can always
  leave", people feel safe staying. Every room made is a file they own
  forever.
- **Receipts travel.** Exported verified PDFs end up in memos and filings —
  quiet little ads that never reveal the room itself.
- **Other tools advertise you.** Once rooms are MCP servers, other apps'
  documentation lists you.
- **Money (later, and only after the gate is fixed):** a one-time
  "Professional" license, working offline, with lawyer-reviewed pages:
  - **Therapists:** "No cloud, no business associate, no BAA needed" —
    they pay $69–150/month today for cloud scribes. Your Whisper +
    `#minutes` is already a session scribe.
  - **Solo lawyers:** "One case = one encrypted file" — the big legal AI
    tools cost $225–639 per user per month. Charge 5–10x less.
  - **Hebrew OCR:** almost no local-AI tools support Hebrew. Israeli
    lawyers, accountants, and immigration cases are a beachhead with zero
    competition. One localized page.
- **Keep writing between launches:** posts about SQLCipher internals,
  making a 4B model reliable, the accuracy numbers. Engineering write-ups
  are the only marketing channel a solo developer can sustain.
- **Free tailwinds:** the new Qwen3.5-4B model could replace your two-model
  vision setup with one model; new M5 Macs make your app much faster for
  free; the Windows port will land exactly when Windows users get local AI
  and want an encrypted container Microsoft won't build.

## 6. Who to sell to first (ranked)

1. **Solo and small-firm lawyers.** About 40% of US law firms are one
   person; solos use Macs the most of any lawyer group. Their tools cost
   hundreds per month. Your pitch: "one case = one encrypted file", and
   quotes that can't be faked. Magic words: privilege, work product,
   Rule 1.6, the Heppner case.
2. **Therapists in private practice.** ~600,000 in the US. They already pay
   for AI note-taking, but every tool is cloud and needs legal paperwork
   (a BAA). Your pitch: "no cloud, no BAA." Ship note templates (SOAP/DAP)
   and one page mapping to HIPAA rules. (Note: your app can't record the
   other side of a video call — so sell it for in-person sessions, don't
   fight Granola.)
3. **Investigative journalists and NGOs.** Little money, loudest voices.
   They refuse cloud AI for sensitive documents. Your Hebrew+English OCR
   and offline transcription fit their work. They'll demand the open spec
   and tool (Bet 3) before they praise you — and their praise unlocks
   groups 1 and 2.

Don't chase M&A data rooms yet — use their crazy prices ($50K+ per deal)
only as a comparison line in your marketing.

## 7. Ideas we killed on purpose (so you don't relitigate them)

The judges attacked all 35 ideas. These died, and why:

- **Cash bounties ("$10K inside an encrypted room — come get it")** — one
  bug in any library you use and the brand is dead. The gibberish-grep demo
  gives the same credibility with no risk.
- **A limited-edition USB stick with the app on it** — mailing a USB stick
  to security-minded people is exactly the thing they're trained to refuse.
  And shipping physical goods is a swamp.
- **"Estate mode" (the room that outlives you)** — a v0.1 app from one
  developer promising to outlive you refutes itself. The recovery sheet
  ships now; inheritance features are for year two.
- **The room working while locked** — impossible without breaking your one
  absolute promise (locked = sealed).
- **Rooms talking to each other, browser-based room opener, new citation
  standards, tamper-proof logs** — version-2 dreams before version 1 has
  users.
- **A Granola-killer meeting notes app** — you can't capture system audio,
  so the first review says "it can't hear the other side of the call."
  Reframed into the therapist wedge (in-person sessions) instead.
- **Paid lawyer/therapist editions at launch** — right audiences, wrong
  order. Free launch first, money after the funnel works.
- **Filming the demo on a real airplane** — same film, minus the plane
  ticket. Airplane mode at a desk proves the same thing.

## 9. More ideas — visual (added later on 2026-07-06)

Inspired by NotebookLM and Obsidian, but with one hard rule: **copy the job,
never the cloud.** Every idea below must run fully offline, and every output
must carry receipts (verified quotes). That's what keeps them "Private Room"
ideas and not clones.

1. **The Room Map (graph view, like Obsidian).** A constellation view of the
   room: every file is a violet star on the ink background; lines connect
   files that talk about the same things. Click a line and it shows you
   *why* — the shared quotes and topics (receipts, again). Memories float as
   small fixed stars; chats link to the files they cited.
   - **Already in the code:** embeddings for every chunk (once the embed
     model downloads — see the gate), so "how similar are these two files"
     is one cosine calculation. The annotation system already knows which
     chats point at which files.
   - **Why it wins:** it's the screenshot. A locked vault is invisible; the
     map makes your private world *look like a world* — without showing any
     actual content in marketing shots (stars can be blurred/renamed).
     Cost: ~2 weeks, mostly frontend (canvas/D3).

2. **The Time Machine (visual history of every AI change).** A timeline
   scrubber per file: every edit shows who did it ("You" / "AI edit" /
   "AI cell change"), a visual before/after diff, and one-click restore.
   - **Already in the code:** version history stores full copies with cause
     labels and restore works (last 10 versions). This is a viewer, not a
     new system.
   - **Why it wins:** "every AI change is reversible, and you can watch it
     happen" is a trust feature no local-AI competitor shows. Cost: ~1 week.

3. **The Front Page (auto-briefing when you unlock).** Opening a room shows
   a calm dashboard before the chat: what changed since last time, the
   newest files, what the AI remembers (memory panel), and three suggested
   questions based on the room's content. Generated fully offline at unlock,
   cached in the room.
   - **Already in the code:** per-file AI summaries are cached (ADD-17),
     memories are editable data, and suggested prompts exist for templates.
   - **Why it wins:** the unlock moment (after The Seal animation) currently
     lands on a cold file list. This makes every return visit feel like the
     room was *waiting for you*. Cost: ~1 week.

4. **The Studio Shelf (one-click artifacts, like NotebookLM's Studio).** A
   small panel with four buttons that turn the room (or selected @files)
   into finished things:
   - **Briefing** — a styled, self-contained HTML report with verified
     quote links back into the sources (the doc-template system and
     #summarize already do 80% of this).
   - **Flashcards & quiz** — the model fills a fixed JSON schema
     (constrained decoding, already built), and a pre-made interactive HTML
     deck runs inside the sandboxed, network-blocked viewer (already
     built). Wrong answer → "explain" pulls the verified source quote.
   - **Mind map** — same trick: schema out of the model, pre-made HTML
     renderer in the sandbox.
   - **Slide deck** — a narrated brief as self-contained HTML slides.
   - **Why it wins:** students, lawyers, and doctors get *finished objects*,
     not chat text — and every object works offline forever because it's a
     self-contained file in the room. The sandboxed HTML runner is the
     secret weapon here; no competitor has one inside an encrypted vault.
     Cost: ~1 week per artifact type; start with flashcards (most viral,
     cheapest).

5. **The Private Podcast (offline audio overview).** One click turns a file
   or folder into a two-host conversation you can listen to — generated and
   spoken entirely on the Mac. The script comes from the local model
   (dialogue as constrained JSON, so it can't go off the rails); the voices
   come from a small local text-to-speech model (e.g. Kokoro ~82M params,
   fast on Metal), bundled exactly the way Whisper already is.
   - **Why it wins:** NotebookLM's most-loved feature, but theirs uploads
     your documents to Google. "A podcast about your secrets that no server
     ever hears" is a headline. Demo: Wi-Fi off, generate a podcast about
     your own medical file, hit play.
   - **Honest cost:** the biggest item here (~3–4 weeks: TTS bundling,
     audio stitching, a player). Ship after the launch, as its own wave —
     it's strong enough to be one.
   - **Skip (for now):** NotebookLM's animated video overviews and
     "talk to the hosts live" — too heavy for one developer; a narrated
     slide deck (Studio Shelf + podcast audio) gets 70% of the effect.

## 10. More ideas — backend (added later on 2026-07-06)

1. **The Airlock (deep research that ends offline).** An opt-in research
   mode: you ask a question, the app fans out web searches, pulls the best
   pages, and **saves each one as an offline copy inside the room** — then
   the web switch turns itself off and the AI answers only from the saved
   copies, with receipts. Material goes IN through the airlock; nothing
   goes out.
   - **Already in the code:** opt-in web search, link import with offline
     page archiving, the web-page cache table, chunking and indexing. This
     is mostly orchestration, not new plumbing.
   - **Why it wins:** it's NotebookLM's Deep Research, but every source
     becomes a file you own forever — and the "goes offline again" ending
     is pure Private Room. Cost: ~2 weeks.

2. **Memory suggestions (automatic memories, with consent).** After a chat,
   the model quietly checks: "did I learn something worth remembering?"
   If yes, a small card appears: *"Remember this? — 'The lease ends March
   2027.'"* One tap saves it to room memory; ignore and it disappears.
   Never writes memory silently.
   - **Already in the code:** the memory system, `add_memory`, and
     constrained decoding for the check. Cost: days, not weeks.

3. **Room roles (custom assistants).** Per-room persona presets that go
   beyond custom instructions: **Tutor** (explains, then quizzes you via the
   Studio Shelf), **Critic** (attacks your draft, with quotes), **Opposing
   counsel** (for the lawyer wedge: red-teams your argument from the other
   side's chair), **Scribe** (for therapists: turns session recordings into
   SOAP/DAP notes). Roles are just curated instruction+command bundles —
   deterministic, cheap, and they map one-to-one to the audience wedges.
   Cost: days per role; the templates system already seeds rooms this way.

4. **Auto-summaries that work for the room, not just files.** The per-file
   summary cache already exists — reuse it: folder descriptions that write
   themselves, "what's in this room" for the Front Page, and smarter
   retrieval (route a question to the right files by their summaries before
   searching chunks). Cost: ~1 week, no new AI machinery.

5. **Big-room scaling (sqlite-vec).** Semantic search today compares every
   chunk one by one — fine for normal rooms, slow for a room holding
   thousands of pages. Adding sqlite-vec (with its new ANN indexes) keeps
   search instant as rooms grow into real archives. Do it when the Airlock
   and clipper features start making rooms big. Cost: ~1 week.

6. **The closet supercomputer (remote inference for power users).** The
   backend already supports a custom Ollama address (built for tests). Make
   it a Settings field: "point this MacBook at the Mac Studio in the
   closet." Your laptop stays light; a bigger model answers over your own
   local network; the privacy story is untouched for everyone else.
   Cost: one Settings field + docs.

7. **Smart import.** When files come in, the model (constrained JSON again)
   suggests a folder, a clean title, and two or three tags — shown as an
   undoable suggestion chip, never applied silently. Makes the "dump 50
   files into the vault" moment feel organized instead of messy.
   Cost: ~1 week.

**Order suggestion for all of section 9–10:** Front Page + Memory
suggestions + Time Machine are cheap trust-builders that can ride the
launch. Room Map and Studio Shelf (flashcards first) are wave two. The
Airlock and the Private Podcast are each strong enough to BE a wave —
don't spend them early.

## 11. Implementation directions — where to start in the code, for every idea

Line numbers are approximate (as of 2026-07-06) — search for the function
name if a line moved.

### The Gate (section 2)

- **The Seal (visual lock ritual).** The "door opens" bloom already exists
  in the create flow (`src/App.tsx:478-589`) — reuse it for unlock. Lock
  lives in the auto-lock machinery (`src/Workspace.tsx:960-992`). Build the
  fold-into-keyhole as a CSS/WebGL overlay in the webview; shrink the window
  with Tauri's window API; play a bundled sound. "Seal & Send" = after lock,
  call macOS `NSSharingServicePicker` from a small Rust command (new file
  next to `biometrics.rs`, which shows the objc2 pattern). Icon art is in
  `art/` with a generation pipeline.
- **Updater.** Three fixes in `src-tauri/tauri.conf.json`: run
  `tauri signer generate`, paste the real pubkey; change the endpoint owner
  `benreich` → `benrben` (also fix the copy in `RELEASING.md`); make the
  release upload `.app.tar.gz` + `latest.json` (the bundler creates them
  once signing is set). Then test one real update v0.1.0 → v0.1.1.
- **Bundled engine.** Follow the whisper.cpp pattern (`src-tauri/src/stt.rs`
  shows how a compiled-in engine + downloaded model works). Add a second
  engine behind the existing engine choice in `src-tauri/src/ollama.rs` /
  `commands.rs` using a llama.cpp binding or a sidecar binary. Model
  download reuses the `pull_model` + `pull-progress` events the first-run
  chooser already uses (`src/Workspace.tsx:313-339`).
- **Wake the sleeping vector search.** One call: `ollama::pull(EMBED_MODEL)`
  during onboarding or first unlock. The backfill loop
  (`backfill_embeddings`, `src-tauri/src/commands.rs:~3569`) already
  notices the model and does the rest.
- **Vision model offer.** Hook the "Ask AI to mark something" bar
  (`src/viewers/ImageView.tsx:49-63`): if no vision model is installed
  (routing check is at `commands.rs:~2892`), show one inline button
  "Download the vision helper (~3 GB)" that calls the same `pull_model`
  flow.
- **CSP.** Set `app.security.csp` in `tauri.conf.json` (start strict:
  `default-src 'self'`; loosen only for blob/data URLs the viewers need —
  the sandboxed `HtmlView` already shows which ones).
- **Wrong "End-to-end encrypted" text.** One string at
  `src/Workspace.tsx:~3347` → "Encrypted on your Mac".
- **Pin crypto + stamp embeddings.** In `src-tauri/src/db.rs`: run explicit
  `PRAGMA cipher_*` settings on open (don't rely on build defaults), and
  write `embed_model` + dimensions into the `meta` table via `migrate()`.
- **Recovery sheet.** Change key handling in `db.rs`: derive the database
  key from a random master key; wrap that master twice (once with the
  password, once with a generated recovery code); store both wraps in
  `meta` via `migrate()`. At create time (`src/App.tsx` create flow), show
  the recovery code once with a "Print / Save PDF" button.
