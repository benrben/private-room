[ideas triage](ideas-triage-2026-07-18.md). Each wave was planned by a dedicated agent
that read the cited code, then adversarially reviewed by an independent critic that
re-read the code hunting for wrong anchors, missed wiring, invariant violations, and
schedule risk. Confirmed findings appear as **Review amendments** after each plan —
executors must apply the amendments as part of the plan; they are not optional notes.

(Claude Code / Codex / Flowed get everything the in-room agent can do, incl. file_pass);
Idea 2 = accepted flagship (dedicated on-screen Workflows page, saveable custom-workflow

`moonshot-impl` carries a large uncommitted working set (the 31-fix security wave, the

before starting new feature branches, or every wave's diff will drag that baggage.

- **[MAJOR]** local_generate's model fallback uses best_default (src-tauri/src/commands/models.rs:16-26), which only filters embedding models — it can return an Ollama `:cloud` model. The plan adds an is_cloud_model check on the room's chat model but then falls back to best_default, which has the same hole: if DEFAULT_MODEL is not installed and the first non-embedding installed model is e.g. minimax-m3:cloud, local_generate silently runs the prompt on Ollama's cloud, betraying the tool's explicit 'runs on the user's LOCAL model' privacy promise. Since it too falls back to the DEFAULT_MODEL *name* when nothing eligible is installed, decide open-decision 3 explicitly: either let the resulting MODEL_MISSING error surface, or return a clear 'no local model installed' error to the agent. Add a test asserting local_generate never resolves to a :cloud model.
- **[MAJOR]** The `long: true` parameter (Job-tier context window) is unimplementable for the no-schema path with the cited API: ollama::generate → plain_generate_body hard-codes num_ctx_for(false, CtxTier::Chat) (ollama.rs:360-375, 390-400); only chat_structured accepts a tier via StructuredOpts (ollama.rs:273-286). - **[MINOR]** The discovery file ~/.private-room/leash.json survives plain app quit: remove_discovery is hooked only to set_room_server off (server.rs:69-72) and teardown_open_room (rooms.rs:306), but Cmd-Q exits through the RunEvent::Exit arm (lib.rs:222-228), which never calls teardown_open_room — it only stops ollama/sidecar and cleans browser previews. ## Idea 7 — Multi-File Atomic Operations (`edit_files`)

**Goal** One batched `edit_files` tool that validates every edit up front and applies all of them in a single `BEGIN IMMEDIATE` transaction, so a five-file refactor either fully lands or fully doesn't — with all batch snapshots sharing a cause tag for group visibility/undo. Follows `set_cells`' validate-everything-then-write shape (agent.rs:1464-1515) and `restore_file_version`'s transaction discipline (safety.rs:32-44).

**Depends on** Idea 4 (uses `compute_edit` and its uniqueness-guarded matcher; shipping batches on top of the old exact-only matcher would multiply the failure rate by N).

**Steps**

1. **(M)** Batch executor in `src-tauri/src/commands/edit_match.rs`:
   ```rust

   pub(crate) fn run_edit_files(conn: &Connection, edits: &[BatchEdit])
       -> Result<BatchApplied, String> // { batch_id, files: Vec<(file_id, real_name, count)> }
   ```
   - Cap: `const MAX_BATCH_EDITS: usize = 20;` reject larger with guidance.
   - **Phase A — validate, zero writes:** iterate edits in order keeping `working: HashMap<file_id, (real_name, Vec<u8>)>`; for each edit resolve via `db::find_file_like`, load bytes from `working` or `db::get_file_bytes`, run Idea 4's `compute_edit` **against the working bytes** (so two edits to the same file chain, exactly like `set_cells` chains `set_cell_in_bytes` over `bytes` at agent.rs:1501-1505), store the result back in `working`. Any failure → `Err(format!("Edit {i} of {n} ({name}): {inner}"))` — nothing written, and the inner message keeps the `closest_snippet`/ambiguity hints so the model can fix just that edit.
   - **Phase B — write:** `batch_id` = first 8 chars of a new UUID; `conn.execute_batch("BEGIN IMMEDIATE")`; for each entry in `working` (one per **file**, final bytes): re-derive text via `extraction::extract_text`, then `store_file_bytes(conn, id, bytes, text, &format!("AI edit (batch {batch_id})"))`; `COMMIT`, on any error `ROLLBACK` and return the error — byte-for-byte the match/rollback shape of safety.rs:38-44. # Wave 4a — Workflows Flagship (Idea 2)

## Idea 2 — LLM Graph Workflows + Scheduler + Flagship Workflows Page

**Goal**: Give Private Room a first-class automation surface: users (and the in-room agent) compose multi-step LLM pipelines from a curated node palette, save them as named reusable workflows, run them manually or on a schedule, and watch them execute as an animated visual pipeline on a dedicated Workflows page. 2. **Compiler `compile_workflow`** (M) — pure, unit-tested, the `build_pass_steps` model (file_pass.rs:76-133): topo-sort nodes → dense `Step` ids `0..n` (every dep lower than its step — this is exactly what keeps the `run_plan` cursor-resume seeding at jobs.rs:135-136 valid), `depends_on` from edges, lane per node from resolved model, `Step.params = { node_id, node_index_map }`. 3. **Generic executor `execute_workflow_step`** (L) — signature mirroring `execute_pass_step` (file_pass.rs:181-191): `(app, job_id, room_path, plan, step, cancel, published)` with the same room-pinning rule (every room access filtered by `room_path`). Two-phase per step:
   - **Liveness check (conditional edges v1 — the precise design)**: `run_plan` (jobs.rs:121-166) is **not modified at all**. Skips are implemented entirely in the executor + artifacts, generalizing `PassArtifact.skipped`: before running node logic, load each incoming edge's parent artifact; an edge is *live* iff the parent artifact is not `skipped` AND (the edge has no `branch` OR the parent is a condition whose artifact `branch` equals the edge's). If NO incoming edge is live (roots are always live), write artifact `{skipped: true}` and return `Ok(())`. Skip therefore propagates transitively through dead subgraphs, join nodes use any-live-in semantics, every step still "completes", `done` stays a dense prefix, and `checkpoint(done.len())` (jobs.rs:162) plus `cursor` keep their exact current meaning. A skipped LocalLlm step costs one no-op wave — acceptable.
   - **Node dispatch**: match on `NodeKind`; inputs = ordered live parents' artifact `result`s. All model calls go through `sidecar_json_cancellable` (as file_pass.rs:241) so Stop drops the in-flight POST; fatal sentinels (`is_fatal`, file_pass.rs:169-171) park the job for Resume, transient errors mark the node's artifact `skipped:true` (honest, resumable). # Wave 4b — Supernatural Voice MVP (Idea 3)

## Idea 3 — Custom Supernatural Voice (MVP: AVSpeechSynthesizer + Web Audio DSP archetypes)

**Goal** Give the room's AI an audible voice: every assistant reply can be spoken aloud through a user-picked "supernatural archetype" (demon / ghost / ancient entity) implemented as synthesis-side pitch+rate settings plus a Web Audio DSP chain at playback, fully on-device with zero model downloads. Audio starts before the full reply lands via sentence-chunked synthesis fed from the existing `ask-delta` stream, with per-message play buttons and a per-room auto-speak toggle. The neural-TTS tier (Piper/Kokoro/ONNX voices) is **explicitly out of scope** — it is a future decision to take only after this DSP MVP proves the feature (triage: Piper's espeak-ng is GPLv3; tract-onnx is validated for TitaNet only, per the warning in `src-tauri/Cargo.toml:58-62`).

**Depends on** none (touches no other idea; engine-agnostic because it taps the `ask-delta` event stream, not the sidecar).

### Synthesis-approach decision (investigated, binding)

**Pick: AVSpeechSynthesizer's write-to-buffer API via `objc2-avf-audio`, not `NSTask say -o`.** Verified feasible: crates.io has `objc2-avf-audio` 0.3.2 whose `AVSpeechSynthesizer` exposes `writeUtterance_toBufferCallback` behind the `AVSpeechSynthesis` feature (checked against docs.rs). Justification over `say -o`:

1. **Pitch control exists only in the API.** `AVSpeechUtterance.pitchMultiplier` (0.5–2.0) is the base layer of every archetype (demon pitches down at synthesis, keeping duration correct); `say` has no pitch flag, which would force all pitch work onto `playbackRate` (pitch and tempo coupled).
2. **No decrypted text/audio on disk.** `say -o` must write a temp file — spoken assistant text is room content, and the app's doctrine is to sweep such artifacts (`commands::cleanup_browser_previews` in `lib.rs:25`). Write-to-buffer keeps PCM in memory end to end.
3. **Latency.** Sentence-chunked synthesis spawns per sentence; a subprocess per sentence adds ~100–300 ms each, in-process synthesis doesn't.
4. **The codebase already owns this exact pattern.** `snapshot.rs:23,72` (block2 `RcBlock` completion handler + `#[cfg(target_os = "macos")]` with a non-mac stub at `snapshot.rs:131`) and `ocr.rs:61-70` establish the objc2 style; `stt_cmds.rs:144` (`transcribe_audio`) establishes base64-audio-over-IPC + `spawn_blocking`.

Known gotchas to code for: buffers arrive as `AVAudioBuffer` on an internal queue → downcast to `AVAudioPCMBuffer`, handle both Float32 and Int16 `commonFormat`; a zero-`frameLength` buffer signals end-of-utterance; keep the `AVSpeechSynthesizer` retained until done and guard with a 30 s watchdog on the mpsc receive. AVSpeechSynthesizer needs no TCC permission or entitlement (unlike mic — `private-room-mic-hardened-runtime` doesn't apply).
